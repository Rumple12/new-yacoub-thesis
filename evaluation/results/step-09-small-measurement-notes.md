# Step 9 small measurement notes

## Environment

- n8n running locally through Docker
- Middleware running locally with `python -m middleware.api.app`
- Deterministic workflow measured through production webhook
- Agent workflow measured through production webhook
- Run count: 1 per mode for initial harness verification

## Commands used

```powershell
python scripts/collect_metrics.py --mode deterministic --webhook-url "http://localhost:5678/webhook/deterministic-baseline" --runs 1
python scripts/collect_metrics.py --mode agent --webhook-url "http://localhost:5678/webhook/agent-minimal" --runs 1
python scripts/aggregate_results.py
```

## Successful measurement evidence

- `evaluation/results/raw/run_20260429T165733Z_deterministic.csv` records successful deterministic high-temperature `fan_on` and low-temperature `fan_off` runs.
- `evaluation/results/raw/run_20260429T165859Z_agent.csv` records successful agent high-temperature `fan_on` and low-temperature `fan_off` runs.
- `evaluation/results/processed/summary_latency.csv`, `summary_resources.csv`, `safety_outcomes.csv`, and `baseline_vs_agent.csv` were generated by `scripts/aggregate_results.py`.

## Setup attempt

`evaluation/results/raw/run_20260429T165702Z_deterministic.csv` records an earlier deterministic setup attempt where the production webhook returned HTTP 404 because the webhook was not registered. This is real setup data and should not be interpreted as part of the successful small-run evidence.
